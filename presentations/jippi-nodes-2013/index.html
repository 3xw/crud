What is CRUD?

I guess everyone know that, but just to make sure

CRUD v2

The Nodes version

  • - One big Component class
  • - Not flexible
  • - Hidden bugs
  • - Almost no documentation
  • - Not (really) maintained
  • - Inconsistent
  • - Not very well unit tested
  • - Bad architecture
  • - Not community backed


It worked really well when I did it

CrudComponent is everything
Can't change out one action for another
Found them while writing v3
Compared to v3 at least
No commits since I left Nodes (~4 months ago)
It behaves different depending on usage
~40% test coverage
"All In One" class, not possible to swap things out
Few people outside Nodes uses it, not marketing for the plugin

CRUD v3

The current version

  • - Separate classes for each Crud action
  • - Very flexible
  • - No known bugs
  • - Tons of documentation
  • - Actively maintained (6 CakePHP core devs)
  • - Very consistent
  • - 90% test coverage
  • - Clean architecture
  • - Community backed


Rewritten from scratch

Everything is isolated classes, uses events to communicate
Everything can be replaced with another implementation
Used in quite a few apps, no bugs
Spend many many weekends on this
May be official scaffolding for Cake 3
No quirks, should behave as expected
~90% test coverage
Observer pattern, emit and listen for events
Big push to make it the foundation for new CakePHP community apps

The Crud Component

Very very skinny

Most logic is moved into Crud Actions & Listeners

Crud Actions

No longer _[index|add|edit|view|delete]Action method in CrudComponent


_${name}Action => Plugin/Crud/Controller/Crud/Action/${name}CrudAction

Crud actions

One class per action

Share nothing

Isolated configuration

Crud actions

  • Index
  • View
  • Add
  • Edit
  • Delete
  • {Your own action here}
